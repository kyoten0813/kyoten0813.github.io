初心者向けGitHub入門

GitHubの基本と他サービスとの違いを分かりやすく解説！

1. GitHubの基本的な使い方

  • リポジトリの作成方法
    1. GitHub（https://github.com/）にアクセスし、アカウントを作成またはログインします。
    2. 画面右上の「＋」ボタンをクリックし、「New repository」を選択します。
    3. リポジトリ名（例: my-first-repo）を入力します。
      • 必要に応じて説明（Description）も入力できます。
      • 「Public」（公開）か「Private」（非公開）を選択します。
      • 「Initialize this repository with a README」にチェックを入れると、最初からREADMEファイルが作成されます。
    4. 「Create repository」ボタンをクリックして作成完了です。

    Tips: リポジトリ名は英数字・ハイフン（-）のみ推奨。後から変更も可能です。

  • ローカルリポジトリの作成とクローン
    1. PCにGitをインストールします（公式サイト）。
    2. コマンドプロンプトやターミナルで、作業用フォルダに移動します。
    3. 既存リポジトリをコピーする場合：
      git clone https://github.com/ユーザー名/リポジトリ名.git
    4. 新規作成の場合：
      git init で空のリポジトリを作成し、git remote add origin ...でGitHubと接続します。

    注意: クローンした場合は、リモート（origin）が自動で設定されます。

  • ファイルの追加・編集・削除
    • 新しいファイルを作成・編集したら、git statusで変更内容を確認します。
    • 変更をステージするには：
      git add ファイル名 または git add .（全て）
    • ファイルを削除した場合も git add 削除したファイル名 でステージします。
  • コミット
    • ステージした変更を履歴として記録するには：
      git commit -m "変更内容の説明"
    • コミットメッセージは簡潔かつ具体的に書くと、後で見返すときに便利です。
    • 例：git commit -m "READMEを追加"
  • プッシュ
    • ローカルのコミットをGitHub上のリポジトリに反映するには：
      git push origin main（mainブランチの場合）
    • 初回のみ git push -u origin main で上流ブランチを設定します。
    • パスワード認証の代わりに「Personal Access Token」を使う場合があります。
  • GitHub Pagesの公開方法
    1. リポジトリの「Settings」タブをクリックします。
    2. 左側メニューから「Pages」を選択します。
    3. 「Branch」で公開したいブランチ（例: main）とフォルダ（/ (root)や/docs）を選びます。
    4. 「Save」をクリックします。
    5. 数分後、「Your site is live at ...」とURLが表示されます。
    • トップページはindex.htmlという名前で配置します。
    • 静的サイト（HTML/CSS/JSのみ）が公開可能です。
    • 独自ドメインの設定も可能です（「Custom domain」欄）。

    注意: 公開まで数分かかる場合があります。リロードして確認しましょう。

  • ブランチとマージ
    • 新しい機能や修正は別ブランチで作業するのが一般的です。
    • ブランチ作成：git checkout -b 新しいブランチ名
    • 作業後、git push origin ブランチ名でリモートにアップロードします。
    • GitHub上で「Pull Request」を作成し、レビュー後に「main」などにマージします。
  • プルリクエスト（Pull Request）
    • 他の人のリポジトリに変更を提案する仕組みです。
    • 自分のリポジトリでも、複数人で開発する場合はPull Requestを使うと履歴が明確になります。
    • レビューやコメント、差分の確認ができます。
  • Issues（イシュー）
    • バグ報告や機能要望、タスク管理に使います。
    • 「New issue」からタイトル・説明・ラベルなどを入力して作成します。
    • 他の人と議論したり、進捗を管理できます。

2. GitHubと他サービスの比較

サービス名 特徴 主な用途
GitHub 世界最大のOSS開発プラットフォーム。
無料で公開リポジトリが作成可能。
GitHub Pagesで静的サイトも公開できる。
  • 豊富なサードパーティ連携（Actions, Codespaces, Copilotなど）
  • スターやフォーク、ウォッチなどSNS的な機能も充実
  • 世界中の開発者とコラボレーション可能
 個人・チームの開発、OSS公開、ポートフォリオ作成
ドキュメント管理、学習用プロジェクト
GitLab 自社サーバーにインストール可能。
CI/CD機能が強力。
  • プライベートリポジトリも無料
  • プロジェクト管理機能（Issue, Board, Wikiなど）
  • オンプレミス運用でセキュリティ重視の企業向け
 企業・組織内の開発、プライベートプロジェクト
セキュリティ要件の高い現場
Bitbucket Atlassian製。
プライベートリポジトリが無料で利用可能。
  • JiraやTrelloなどAtlassian製品との連携が強力
  • 小規模チーム向けの無料枠が充実
 小規模チーム、Jira連携
プロジェクト管理とソース管理の統合

比較まとめ

  • GitHubはOSSや個人開発、学習用途に最適。世界中の開発者と交流できる。
  • GitLabは企業や組織での利用、CI/CD自動化に強み。
  • BitbucketはAtlassian製品との連携や小規模チームに向いている。

どのサービスもGitをベースにしているため、基本操作は共通です。
用途や規模、必要な機能に応じて使い分けましょう。

3. まとめ

GitHubは初心者にも使いやすく、学習やポートフォリオ作成にも最適です。
まずはリポジトリを作成し、ファイルをアップロードしてみましょう。
分からないことがあれば、公式ドキュメントやコミュニティも活用してみてください！

よくある質問（FAQ）

  • Q: GitHubは無料で使えますか？
    A: 公開リポジトリは無料、プライベートリポジトリも無料枠があります。
  • Q: プログラミング初心者でも使えますか？
    A: はい。GUIツール（GitHub Desktopなど）もあり、コマンドが苦手でも始められます。
  • Q: 共同開発はどうやって行うの？
    A: Pull RequestやIssue、Wiki、Projectsなどの機能を使って複数人で効率的に開発できます。
  • Q: 間違えてファイルを消した場合は？
    A: Gitの履歴から過去の状態に戻すことができます（git checkoutやgit revertなど）。

注意点・トラブルシューティング

  • ファイルサイズが100MBを超えるとアップロードできません（LFSを利用）。
  • パスワード認証が廃止され、Personal Access Tokenが必要な場合があります。
  • 公開リポジトリは誰でも閲覧できるため、個人情報や秘密情報は絶対にアップしないでください。
  • コミットやプッシュ時にエラーが出た場合は、エラーメッセージをよく読み、公式ヘルプを参照しましょう。

一歩進んだ使い方

  • GitHub Actionsで自動テストやデプロイを設定する
  • READMEにバッジや画像、リンクを追加して見やすくする
  • IssueテンプレートやPull Requestテンプレートを活用する
  • GitHub Pagesでポートフォリオやブログを公開する

まずは小さなプロジェクトから始めて、少しずつ慣れていきましょう。
GitHubを使いこなせば、世界中の開発者とつながるチャンスが広がります！
